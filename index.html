LESSON 1 -THE BASICS OF HTML

World Wide Web

World Wide Web is hypertext system that operates over the Internet. It's a way for the computers to communicate and to share information. When someone visits a webpage the computes he uses sends an HTML request to a server, a computer that host the files that make up the web. The servers then finds the requested HTML document and sends it to the user's computer.

Web Browser

A Web Browser is a program that runs on your computer to display files found on the web. To see a video on what the world wide web really is and how the web browser works please click on the following. Video

HTML

HTML means Hypertext MarkUp Language . World Wide Web contains mostly HTML documents. HyperText defines the text content of the document, so what we actually see. And the MarkUp describes how the text content looks.

Tags and Elements

HTML documents are made of HTML elements. The Basic HTML element have an opening and a closing tag. When we write HTML we use tags in order to tell the browsers the type of each element.

HTML Attributes

Some Elements accept additional values called attributes in theis opening tags. Attributes provide additional information to the browser. An example is the href attribute which is used in compination with the a tag to create hyperlink to a webpage.

Inline Vs Block Elements

HTML elements can be either inline on block elements. The difference between these two is that the block elements create an invisible box around their content.

Container Elements

They are Elements that do nothing but contain stuff inside themselves. These Elements are used to create webpages with complex structures. An inline Container Element is span. Lets test it. I am just testing the span element. A Block Container Element is div. Lets check that too.

LESSON 4: ADDING CSS FOR STYLE

What is the CSS?

CSS means Cascading Style Sheets and it's the code written to control the style of your HTML. With CSS we can actually apply rules to every element.

Order in HTML matters. Styles can be defined in different places and are applied in the following order,with definitions further down the list overwriting previous definitions:

a) the default style of a browser
b) stylesheet in a separate file which will be CSS ( this is what you will be mostly using)
c) stylesheet inside HTML
d) inline style in an element ( this can be done but it should be avoided)


Cascading means that rules are applied not only to the elements they directly match, but also to all of those elements' child elements. However, if a child element has multiple overlapping rules defined for it, the more specific rule takes effect.

CSS Vocabulary

(a)Rule: A line of CSS code describing the value that a certain attribute should take.
(b) Property: The property you want to change.
(c) Value: The value you want to assign to the attribute.
(d) Selector: The name you use in order to target the elements that are assigned to a calss or an id attribute in the html.
(e) Class: A class refers to a group of elements that can be styled together. Class names should not contain spaces or any other punctuation marks .
(f) IDs : IDs are unique identifiers that uniquely identifies an html.

How to write CSS

In CSS you first write a selector , and this defines to which elements in the page a particular style will apply to.

Tag Selectors

You can either select an HTML tags like p tag for example, and you can set a style which will apply to every p tag in your HTML document. To do this you have to write in your CSS code the following

p{
....
}

Class Selectors

Another way to apply a style is to select elements by class.A class refers to a group of elements that can be styled together. To define and to style a class in CSS you need to write the following.

.classname{
.......
}
Class selectors are your friend. They are probably the most usefull selectors out there. In part because they are well supported in all browsers. In part because you can add multiple classes just separated by a space on HTML elements .

ID Selectors

You can also use ID Selectors.IDs are unique identifiers that uniquely identifies and HTML Element.To define an ID Selector in CSS you need to write this :
#IDNAME{
.......
}
Note that the ID selectors are the most powerfull type of selector in terms of CSS specificity. Meaning that they beatout other type of selectors and the styles within win. It's better to have lower specificity selectors that are easier to overwrite when needed.

4) Add a link tag inside the head element. Since link is a void element you don't need to add a closing tag.
5) Add the following attributes to your link :

The Box Revisited
HTML are boxes and each box has 4 components

1) Content is the image or text that is appearing on the website. The other layers serve to protect it so that other items on the pages don't overlap or sit too close to it.
2) Patting clears the area around the content, and its affected by the background color of the box.
3) Border . The border's inherited from the color property of the box.
4) Margin is the outer layer.This does not have any background color and it's completetly transparent. It's the space between boxes.

Box Positioning
Divs are block elements so by default thay take up the entire width of the page. There are many things you can do in CSS to position boxes. And it's probably one of the most complicated things you can do in CSS. You can use Flexbox. Flexbox means flexible box and it provides an efficient way to lay out, align and distribute space among items in a container or div. Adding the rule display:flex; to the aproppriate CSS will overwrite the behavior and lets DIVs (boxes) appear next to each other.

Thus in your CSS you need to write the following:

.examplename{
display:flex;
}

In the above example we grouped some elements under a class. The property we changed under this class is the display and we set this to the value flex. By doing so the boxes (that are grouped under this class) will be alligned next to each other in our website .
